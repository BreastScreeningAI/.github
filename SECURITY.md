# Security Policy

## Supported Projects

Security and privacy reports are reviewed for actively maintained repositories
in the BreastScreening-AI organization.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security, privacy, or data exposure
concerns.

If GitHub private vulnerability reporting is enabled on the affected repository,
use that channel. Otherwise, contact the repository maintainers directly through
the contact details published in the relevant repository or on the public
project website.

## What To Include

- The affected repository, file, endpoint, or page.
- A concise description of the issue.
- Steps to reproduce or evidence of the problem.
- Any suspected exposure of credentials, private data, or patient-identifiable
  information.
- Your preferred contact details for follow-up.

## Data Handling

Do not include patient-identifiable data, private datasets, credentials, tokens,
or secrets in reports unless the maintainers explicitly ask for a secure
transfer method.
